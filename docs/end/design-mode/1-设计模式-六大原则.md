# 笔记一 设计模式——六大原则

## <font color=42b983>1. 开闭原则 （Open Close Principle）</font>

- 对扩展开放，对修改关闭。

> 1. 在程序需要进行扩展的时候，不能去修改原有的代码，实现热插拔的效果。
> 2. 为了使程序的扩展性好，易于维护和升级。
> 3. 想要达到这样的效果，我们需要使用接口和抽象类。
